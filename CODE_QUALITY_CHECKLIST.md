# ✅ 代码质量检查清单

## 📋 检查清单

### 代码规范

#### 命名约定
- [ ] 类名使用大驼峰命名 (PascalCase)
- [ ] 函数和变量使用小驼峰命名 (camelCase)
- [ ] 常量使用大写下划线命名 (UPPER_SNAKE_CASE)
- [ ] 私有属性使用下划线前缀
- [ ] 文件名使用小写，单词间用连字符分隔

#### 注释规范
- [ ] 所有公共函数都有 JSDoc 注释
- [ ] 注释包含参数说明 (@param)
- [ ] 注释包含返回值说明 (@returns)
- [ ] 复杂逻辑有清晰的解释
- [ ] 避免无意义的注释

#### 代码格式
- [ ] 使用 4 个空格缩进
- [ ] 行宽不超过 100 字符
- [ ] 函数不超过 50 行
- [ ] 文件不超过 500 行
- [ ] 使用分号结尾

## 🔍 自检问题

### 代码审查时问自己

1. **可读性**
   - 这段代码容易理解吗？
   - 变量命名是否清晰？
   - 函数是否太长？
   - 需要添加注释吗？

2. **可维护性**
   - 代码容易修改吗？
   - 有重复代码吗？
   - 依赖关系清晰吗？
   - 符合单一职责吗？

3. **性能**
   - 有性能瓶颈吗？
   - 需要优化吗？
   - 有内存泄漏风险吗？
   - 资源使用合理吗？

4. **测试**
   - 代码有测试吗？
   - 测试覆盖全面吗？
   - 测试容易理解吗？
   - 测试稳定吗？

5. **安全**
   - 有安全风险吗？
   - 输入验证了吗？
   - 错误处理了吗？
   - 数据保护了吗？
