# 📋 代码质量检查清单

## 一、安全检查

### 1.1 XSS 防护
- [ ] 所有 `innerHTML` 使用是否经过安全审查
- [ ] 用户输入是否使用 `escapeHtml()` 转义
- [ ] 动态内容是否使用 `safeTemplate()` 或 `textContent`
- [ ] 外部数据是否验证后再渲染

### 1.2 输入验证
- [ ] 所有用户输入是否经过验证
- [ ] 数字输入是否检查 NaN
- [ ] 字符串输入是否限制长度
- [ ] 数组索引是否检查范围

### 1.3 数据存储
- [ ] localStorage 操作是否有错误处理
- [ ] 存储失败是否有用户提示
- [ ] 敏感数据是否加密存储
- [ ] 数据恢复机制是否完善

---

## 二、性能检查

### 2.1 内存管理
- [ ] 定时器是否有清理方法
- [ ] 事件监听器是否有移除方法
- [ ] 动画帧是否有取消方法
- [ ] 大对象是否有释放机制

### 2.2 DOM 操作
- [ ] 是否避免频繁 innerHTML
- [ ] 是否使用文档片段批量操作
- [ ] 是否缓存 DOM 查询结果
- [ ] 是否使用事件委托

### 2.3 算法效率
- [ ] 循环是否有优化空间
- [ ] 是否使用缓存避免重复计算
- [ ] 是否使用节流/防抖
- [ ] 是否避免同步阻塞操作

---

## 三、代码规范

### 3.1 命名规范
- [ ] 变量名是否使用 camelCase
- [ ] 常量名是否使用 UPPER_SNAKE_CASE
- [ ] 类名是否使用 PascalCase
- [ ] 文件名是否使用 kebab-case

### 3.2 注释规范
- [ ] 函数是否有 JSDoc 注释
- [ ] 复杂逻辑是否有说明注释
- [ ] 算法是否有时间复杂度说明
- [ ] 特殊处理是否有原因说明

### 3.3 结构规范
- [ ] 函数长度是否超过 100 行
- [ ] 是否有重复代码需要提取
- [ ] 魔法数字是否提取为常量
- [ ] 是否遵循单一职责原则

---

## 四、错误处理

### 4.1 异常捕获
- [ ] 是否使用 try-catch 处理可能失败的操作
- [ ] 是否有全局错误处理机制
- [ ] 是否记录错误日志
- [ ] 是否有用户友好的错误提示

### 4.2 边界条件
- [ ] 是否处理空值/undefined
- [ ] 是否处理数组越界
- [ ] 是否处理除零情况
- [ ] 是否处理网络错误

### 4.3 用户反馈
- [ ] 是否使用统一通知系统
- [ ] 是否避免使用 alert()
- [ ] 错误信息是否清晰易懂
- [ ] 是否提供恢复建议
